# Batch processing in BMC AMI Apptune for Db2
## Test Update
Many Db2 performance problems do not produce obvious real-time symptoms.

Often, to make the diagnosis you need a detailed analysis using data obtained over long periods of time. Where a large volume of data exists to analyze, a printed report is easier to read than an online report that requires scrolling through many screens.

The System and SQL Performance for DB2 products address this situation by enabling you to produce printed reports from a batch job.

The System and SQL Performance for DB2 products provide a full set of online reports, some of which can be run in batch.

Other reports are designed specifically for batch reporting.

You can produce batch reports from archived or active log file data sets and EXPORT data sets.

Use the DOMBRPT1 utility to perform batch processing.
